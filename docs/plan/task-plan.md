## 2. 태스크별 설명

### TASK-001: Nuxt 3 프로젝트 초기화 및 기본 설정

- **관련 기능:** 공통
- **설명:** Nuxt 3 + TypeScript 프로젝트를 생성하고, ESLint, Prettier, Tailwind CSS, Prisma 등 개발 도구를 설정한다. `.env` 파일 구조 및 환경변수 관리를 포함한다.
- **완료 조건:**
  - `npx nuxi init` 기반 프로젝트가 정상 실행된다
  - TypeScript 설정이 완료되어 타입 체크가 동작한다
  - 개발 서버(`npm run dev`)가 정상 실행된다
  - `.env.example`에 필요한 환경변수가 정의되어 있다
- **담당 역할:** 풀스택

### TASK-002: Prisma + Turso(libSQL) 데이터베이스 스키마 설계 및 마이그레이션

- **관련 기능:** 공통
- **설명:** User, Team, Session, Feedback, Insight, ActionItem, Vote 등 핵심 테이블의 Prisma 스키마를 설계한다. 3단계 권한(ADMIN/LEADER/MEMBER) 역할 enum을 포함한다. 로컬 개발은 SQLite 파일, 배포는 Turso(libSQL)를 사용한다. ActionItem은 Sprint 2에서 `sessionId`, `issueIndex` 필드가 추가됐다 (쿼리 단순화 목적).
- **완료 조건:**
  - Prisma 스키마 파일(`schema.prisma`)이 작성되어 있다
  - 로컬: `npx prisma db push`로 스키마 적용, Turso: `npm run db:turso:push`로 적용
  - `@prisma/adapter-libsql`을 통한 런타임 Turso 연결이 동작한다
  - Seed 데이터(최고관리자 계정, 샘플 팀)가 포함된다
  - ActionItem에 `sessionId`(세션 직접 참조), `issueIndex`(인사이트 이슈 인덱스) 포함
- **담당 역할:** 백엔드

### TASK-003: Tailwind CSS 설정 및 공통 레이아웃/GNB 구현

- **관련 기능:** 공통
- **설명:** Tailwind CSS를 설정하고, GNB(상단 네비게이션)를 포함한 공통 레이아웃을 구현한다. GNB에는 로고, 홈, 공유인사이트(P1), 관리자(최고관리자 전용), 사용자 이름, 로그아웃 버튼이 포함된다. 권한별 메뉴 노출 제어를 포함한다.
- **완료 조건:**
  - Tailwind CSS가 정상 동작한다
  - 공통 레이아웃에 GNB가 표시된다
  - 사용자 역할에 따라 GNB 메뉴가 동적으로 표시된다
  - 모바일 반응형이 기본 적용된다
- **담당 역할:** 프론트엔드

### TASK-004: 공통 UI 컴포넌트 구현

- **관련 기능:** 공통
- **설명:** 버튼(Primary/Secondary/Danger), 입력 필드, 모달, 토스트 알림, 뱃지, 확인 다이얼로그, 로딩 스피너, 빈 상태 안내 등 화면 기획서의 공통 UI 요소를 재사용 가능한 컴포넌트로 구현한다.
- **완료 조건:**
  - 각 공통 컴포넌트가 독립적으로 동작한다
  - Props를 통해 변형(variant)을 제어할 수 있다
  - 전체 화면에서 일관된 디자인이 적용된다
- **담당 역할:** 프론트엔드

### TASK-005: 회원가입 API 엔드포인트 구현

- **관련 기능:** FEAT-000
- **설명:** 이메일, 비밀번호, 이름을 받아 신규 계정을 생성하는 API를 구현한다. 비밀번호 해싱(bcrypt), 이메일 중복 검사, 비밀번호 유효성 검사(최소 8자, 영문+숫자)를 포함한다. 기본 역할은 MEMBER, 팀 미배정 상태로 생성한다.
- **완료 조건:**
  - `POST /api/auth/register` 엔드포인트가 동작한다
  - 비밀번호가 bcrypt로 해싱되어 저장된다
  - 이메일 중복 시 적절한 에러 메시지가 반환된다
  - 비밀번호 규칙 위반 시 유효성 검사 에러가 반환된다
- **담당 역할:** 백엔드

### TASK-006: 로그인/로그아웃 API 및 JWT 세션 관리 구현

- **관련 기능:** FEAT-000
- **설명:** 이메일/비밀번호 기반 로그인 API, JWT 토큰 발급 및 검증, 토큰 자동 만료, 로그아웃 처리를 구현한다.
- **완료 조건:**
  - `POST /api/auth/login` 엔드포인트가 동작한다
  - JWT 토큰이 발급되고 자동 만료된다
  - `POST /api/auth/logout` 엔드포인트가 동작한다
  - 인증되지 않은 요청이 401 에러로 거부된다
- **담당 역할:** 백엔드

### TASK-007: 3단계 권한(ADMIN/LEADER/MEMBER) 미들웨어 구현

- **관련 기능:** FEAT-000
- **설명:** API 요청 시 사용자 역할을 검증하는 서버 미들웨어를 구현한다. 각 API 엔드포인트에 필요한 최소 권한을 설정할 수 있도록 한다.
- **완료 조건:**
  - 역할 기반 접근 제어가 모든 보호된 API에 적용된다
  - ADMIN은 모든 기능에 접근 가능하다
  - LEADER는 팀 관련 관리 기능에 접근 가능하다
  - MEMBER는 기본 기능에만 접근 가능하다
  - 권한 부족 시 403 에러가 반환된다
- **담당 역할:** 백엔드

### TASK-008: 로그인 화면(SCR-000) 구현

- **관련 기능:** FEAT-000
- **설명:** 이메일/비밀번호 입력 폼, 실시간 유효성 검사, 로그인 버튼, 에러 메시지 표시, 회원가입 링크를 포함한 로그인 화면을 구현한다.
- **완료 조건:**
  - 이메일 형식 실시간 유효성 검사가 동작한다
  - 로그인 성공 시 홈(팀 배정) 또는 팀 합류(팀 미배정) 화면으로 이동한다
  - 로그인 실패 시 에러 메시지가 표시된다
  - 회원가입 링크가 동작한다
- **담당 역할:** 프론트엔드

- **설명:** 대시보드 피드백 카드에 투표 버튼과 투표 수를 표시하고, 투표순/최신순 정렬 드롭다운을 구현한다.
- **완료 조건:**
  - 피드백 카드에 투표 버튼과 투표 수가 표시된다
  - 투표/취소 토글이 즉시 반영된다
  - 투표순/최신순 정렬이 동작한다
  - 이미 투표한 피드백은 버튼이 활성 상태로 표시된다
- **담당 역할:** 프론트엔드

### TASK-030: 인사이트 공유/비공유 토글 API 구현 (팀장 권한)

- **관련 기능:** FEAT-007

- **완료 조건:**
  - `PATCH /api/insights/:id/share` 엔드포인트가 동작한다 (팀장 이상)
  - 공유 상태 변경이 DB에 반영된다
  - 공유된 인사이트는 모든 사용자가 조회 가능하다
  - 공유 시 피드백 원문은 비공개된다
- **담당 역할:** 백엔드

### TASK-031: 공유된 인사이트 목록 화면(SCR-005) 구현

- **관련 기능:** FEAT-007
- **설명:** 다른 팀이 공유한 인사이트를 카드 목록으로 표시하는 화면을 구현한다. 팀명, 세션명, 요약 미리보기, 클릭 시 상세 펼침을 포함한다.
- **완료 조건:**
  - 공유된 인사이트가 카드 형태로 표시된다
  - 카드에 팀명, 세션명, 요약 미리보기가 표시된다
  - 카드 클릭 시 상세 내용(핵심 이슈, 권장 액션)이 펼쳐진다
  - 피드백 원문은 표시되지 않는다
  - 공유된 인사이트가 없을 때 빈 상태 안내가 표시된다
- **담당 역할:** 프론트엔드
