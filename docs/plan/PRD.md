# RetroLens PRD

## 1. 개요

- **프로젝트 한 줄 소개:** 팀 프로젝트 회고를 수집하고, AI가 패턴을 분석해 실행 가능한 인사이트를 제공하는 도구
- **작성일:** 2026-03-13
- **버전:** 1.2.0
- **상태:** Draft

---

## 2. 문제 정의

### 2.1 현재 상황 (As-Is)

팀 프로젝트 회고는 애자일/스크럼 팀의 필수 활동이지만, 대부분의 팀에서 다음과 같은 문제를 겪고 있습니다:

| 문제 | 구체적 상황 | 비즈니스 영향 |
|------|-------------|---------------|
| **형식적인 회고** | "좋았어요", "다음엔 잘해요"로 끝남 | 실질적 개선 없이 동일 문제 반복 |
| **솔직한 의견 부재** | 대면/실명 환경에서 부정적 피드백을 꺼림 | 진짜 문제가 수면 위로 드러나지 않음 |
| **기록 유실** | 회고 결과가 노션/슬랙/문서에 흩어짐 | 과거 인사이트 활용 불가 |
| **액션 아이템 미추적** | "다음엔 이렇게 하자" -> 다음 스프린트에 잊힘 | 회고의 실효성 저하 |
| **패턴 분석 부재** | 동일 문제가 반복되는지 파악 어려움 | 근본 원인 미해결, 팀 번아웃 |

## 3. 목표 사용자

### 3.1 사용자 유형별 역할/니즈

**3단계 권한:**

| 사용자 유형 | 역할 | 니즈 |
|-------------|------|------|
| **최고관리자** | 사내 전체 시스템 관리 | 팀 생성/관리, 전체 통계, 사용자 권한 관리 |
| **팀장** | 팀 회고 진행 및 결과 관리 | 회고 생성, 인사이트 생성/공유, 팀원 관리 |
| **팀원** | 회고 참여자 | 본인 팀 피드백/투표, 공유된 다른 팀 인사이트 조회 |

### 3.2 접근 권한 매트릭스

| 대상 | 팀원 | 팀장 | 최고관리자 |
|------|------|------|-----------|

| 본인 팀 투표 | O | O | O |
| 본인 팀 대시보드 | O | O | O |
| 본인 팀 인사이트 생성 | - | O | O |

### 3.3 사용자 페르소나

**페르소나 1: 김팀장 (팀장)**
- 8명 규모 개발팀 운영, 팀 생성 및 팀원 관리 담당
- 매 스프린트 회고 진행, 인사이트 생성 후 다른 팀과 공유
- "우리 팀 회고 결과를 다른 팀과 공유해서 서로 배우고 싶어"

**페르소나 2: 이개발 (팀원)**
- 입사 6개월, 팀 회고에서 의견 내기 부담스러움
- "배포 프로세스 문제 말하고 싶은데, 선배가 담당이라 말하기 애매해..."

## 4. 목표

### 4.1 제품 목표

| 목표 | 측정 지표 | 목표값 |
|------|----------|--------|
| 솔직한 피드백 유도 | 회고당 평균 피드백 수 | 평균 15개 이상 |
| 인사이트 자동화 | AI 요약 생성 시간 | 30초 이내 |
| 액션 아이템 추적 | 액션 아이템 완료율 | 70% 이상 |

### 4.2 프로젝트 목표 (해커톤)

- **MVP 완성:** 2일 내 핵심 기능 동작하는 서비스 배포
- **문서화 완성:** PRD, README, CLAUDE.md 완비
- **테스트 커버리지:** 핵심 비즈니스 로직 50% 이상

### 4.3 핵심 가치 제안

1. **익명성 보장** -> 심리적 안전감 확보 -> 솔직한 피드백
2. **AI 문체 변환** -> 문체 기반 식별 차단 -> 진정한 익명성
3. **AI 패턴 분석** -> 반복 문제 자동 발견 -> 근본 원인 해결
4. **시각화 대시보드** -> 팀 건강도 한눈에 -> 빠른 의사결정

## 5. 핵심 기능

### 5.1 기능 우선순위 정의

| 우선순위 | 설명 | 해커톤 범위 |
|----------|------|-------------|
| **P0** | MVP 필수 기능 | 반드시 구현 |
| **P1** | 핵심 사용성 향상 + 최고관리자 | 시간 여유 시 |
| **P2** | 향후 확장 기능 | 해커톤 이후 |

### 5.2 P0 기능 (MVP)

> **P0 선정 근거:** 로그인 -> 익명 피드백 수집(+ AI 문체 변환) -> 대시보드 확인 -> AI 인사이트 도출이라는 핵심 사용자 플로우를 완성하는 데 필수적인 6개 기능입니다.

#### FEAT-000: 인증 (로그인/회원가입)

| 항목 | 내용 |
|------|------|
| **설명** | 사용자 인증 및 계정 관리 |
| **인증 방식** | 이메일 + 비밀번호 (MVP) |
| **세션 관리** | 인증 후 세션 유지 (자동 만료 포함) |

**회원가입 플로우:**
```
사용자 회원가입 (이메일, 비밀번호, 이름)
       |
       v
계정 생성 (기본 역할: 팀원, 팀 미배정)
       |
       v
팀장이 팀에 초대 또는 팀 코드로 합류
       |
       v
서비스 이용 가능
```

| 기능 | 설명 | MVP 포함 |
|------|------|----------|
| 회원가입 | 이메일, 비밀번호, 이름 입력 | O |
| 로그인 | 이메일 + 비밀번호 | O |
| 로그아웃 | 세션 종료 | O |
| 비밀번호 유효성 | 최소 8자, 영문+숫자 | O |
| 비밀번호 찾기 | 이메일 인증 후 재설정 | - |
| 소셜 로그인 | Google, GitHub 등 | - |
| 이메일 인증 | 가입 시 이메일 확인 | - |

**완료 조건:**
- 회원가입 후 로그인 가능
- 인증 후 세션 토큰 발급
- 인증된 사용자만 API 접근 가능
- 팀 미배정 사용자는 팀 합류 화면만 접근 가능
- 3단계 권한 구현: 최고관리자/팀장/팀원

#### FEAT-001: 회고 세션 생성

| 항목 | 내용 |
|------|------|
| **설명** | 새로운 회고 세션을 생성하고 팀원들이 접근할 수 있는 링크 제공 |
| **사용자 스토리** | 스크럼 마스터로서, 스프린트 종료 후 회고 세션을 생성하여 팀원들에게 공유하고 싶다 |
| **입력** | 세션 제목, 프로젝트명, 회고 기간(선택) |
| **출력** | 고유 세션 URL, 세션 상세 페이지 |

**완료 조건:**
- 세션 생성 후 고유 URL 발급
- URL로 접근 시 피드백 입력 가능
- 세션 목록에서 조회 가능
- 세션 종료(마감) 기능 제공 (팀장 이상)

#### FEAT-002: 익명 피드백 입력

| 항목 | 내용 |
|------|------|
| **설명** | K/P/T(Keep/Problem/Try) 형식으로 익명 피드백 제출 |
| **사용자 스토리** | 팀원으로서, 누가 작성했는지 알 수 없는 환경에서 솔직한 피드백을 제출하고 싶다 |
| **입력** | 카테고리(Keep/Problem/Try), 피드백 내용 |
| **출력** | 제출 완료 확인 |

**완료 조건:**
- 로그인한 팀원만 본인 팀 세션에 피드백 제출 가능 (접근은 인증 기반, 저장은 익명)
- 피드백 저장 시 작성자 정보를 연결하지 않음 (서버에서도 추적 불가)
- 카테고리별 다중 피드백 입력 가능
- 익명성 보장을 위해 제출 후 수정/삭제 불가

#### FEAT-002-1: AI 문체 변환 (익명성 강화)

| 항목 | 내용 |
|------|------|
| **설명** | 피드백 제출 시 AI가 문체를 중립적으로 변환하여 작성자 특정 방지 |
| **사용자 스토리** | 팀원으로서, 내 말투나 표현 습관으로 누군지 들킬까봐 걱정되는데, AI가 문체를 바꿔줬으면 좋겠다 |
| **입력** | 원본 피드백 텍스트, 문체 변환 옵션(ON/OFF) |
| **출력** | 변환된 피드백 텍스트 (미리보기), 최종 제출 |

**완료 조건:**
- 문체 변환 토글 옵션 제공
- 변환 전/후 비교 미리보기
- AI가 의미를 유지하면서 개인 특유의 표현을 중립화
- 변환 후 사용자 확인 후 제출

#### FEAT-004: 팀 대시보드

| 항목 | 내용 |
|------|------|
| **설명** | 회고 세션의 전체 피드백을 카테고리별로 조회 |
| **사용자 스토리** | 팀 전체로서, 수집된 모든 피드백을 한눈에 보고 논의하고 싶다 |
| **입력** | 세션 ID |
| **출력** | Keep/Problem/Try 카테고리별 피드백 목록 |

**완료 조건:**
- 카테고리별 탭 또는 섹션 구분
- 피드백 내용 전체 표시
- 피드백 수 카운트 표시

#### FEAT-005: AI 인사이트 생성

| 항목 | 내용 |
|------|------|
| **설명** | 수집된 피드백을 AI가 분석하여 핵심 이슈와 권장 액션 도출 |
| **사용자 스토리** | 스크럼 마스터로서, 피드백을 일일이 분석하지 않고 핵심 인사이트를 빠르게 파악하고 싶다 |
| **입력** | 세션의 전체 피드백 |
| **출력** | 요약문, 핵심 이슈 목록, 권장 액션 |

**완료 조건:**
- "인사이트 생성" 버튼 클릭 시 AI 분석 실행
- 30초 이내 결과 표시
- 핵심 이슈 3~5개 추출
- 각 이슈에 대한 권장 액션 제시

## 10. 차별화 포인트

### 10.1 핵심 차별점: AI 문체 변환을 통한 진정한 익명성

기존 익명 회고 도구들의 근본적 한계:

| 문제 | 설명 |
|------|------|
| **문체 기반 식별** | 특정 말투, 이모지 패턴 등으로 작성자 추정 가능 |
| **소규모 팀 취약성** | 5~10명 팀에서는 말투만으로 70% 이상 식별 가능 |
| **심리적 안전감 미확보** | "익명"이라도 들킬까 봐 솔직한 피드백 꺼림 |

**RetroLens의 해결책:**
```
[원본] "배포할 때마다 30분씩 걸리는 거 진짜 미치겠음... ㅠㅠ"
         | AI 문체 변환
[변환] "배포 프로세스에 약 30분이 소요되어 개선이 필요합니다."
```

- AI가 의미는 유지하면서 개인 특유의 말투/이모지/표현을 중립화
- 사용자가 변환 결과를 확인하고 재요청 가능
- **기술적으로 익명성을 보장하는 유일한 도구**

### 10.2 경쟁 도구 비교

| 기존 도구 | 한계점 | RetroLens 차별점 |
|-----------|--------|------------------|
| **모든 익명 도구** | 문체/말투로 작성자 추정 가능 | **AI 문체 변환으로 진정한 익명성** |
| **Miro/FigJam** | 수동 정리, 분석 기능 없음 | AI 자동 요약 + 핵심 이슈 추출 |
| **Retrium** | 월 $29+, 기능 과다로 학습 비용 높음 | 오픈소스 (해커톤 MVP) + 익숙한 K/P/T 프레임워크 |
| **Google Forms** | 분석 없음, 회고 전용 UX 부재 | 실시간 인사이트 + 전용 대시보드 |
| **Notion 템플릿** | 매번 수동 설정, 히스토리 분산 | 원클릭 세션 생성 + 통합 히스토리 |

### 10.3 단순함의 가치

| 설계 원칙 | 이유 |
|-----------|------|
| **K/P/T 프레임워크 유지** | 대부분의 팀이 이미 익숙함 -> 학습 비용 제로 |
| **핵심 기능만 집중** | 기능이 많으면 오히려 사용 안 함 |
| **3클릭 내 완료** | 세션 생성 -> 피드백 입력 -> 인사이트 확인 |

| **가용성** | 클라우드 플랫폼 기본 가용성 (99.9%) |

### 11.1 익명성 정책

| 기능 | 익명 여부 | 원칙 |
|------|----------|------|
| **피드백 작성** | 익명 | 피드백과 작성자 정보를 기술적으로 연결하지 않음 |
| **AI 문체 변환** | 익명 강화 | 개인 특유 문체 제거 |
| **참여 확인** | 분리 | 참여 기록과 피드백을 연결하지 않음 |
| **투표** | 실명 | 공개된 피드백에 대한 공감 표시이므로 사용자 ID 기록 |

**피드백 수정/삭제 정책:** 익명성 보장을 위해 제출 후 수정/삭제 불가합니다.
