## 2. 화면별 구성 요소

### SCR-000: 로그인

- **목적**: 기존 사용자가 이메일과 비밀번호로 인증하여 서비스에 접근한다.
- **진입 경로**: 비로그인 상태에서 서비스 URL 접근 시 자동 리다이렉트 / 회원가입 완료 후

**와이어프레임:**
```
┌─────────────────────────────────────┐
│                                     │
│           🔍 RetroLens              │
│                                     │
│  ┌─────────────────────────────┐    │
│  │ 📧 이메일                    │    │
│  └─────────────────────────────┘    │
│  ┌─────────────────────────────┐    │
│  │ 🔒 비밀번호                  │    │
│  └─────────────────────────────┘    │
│                                     │
│  ┌─────────────────────────────┐    │
│  │         로그인               │    │
│  └─────────────────────────────┘    │
│                                     │
│  ⚠️ 에러 메시지 영역 (조건부)       │
│                                     │
│   계정이 없으신가요? [회원가입]      │
│                                     │
└─────────────────────────────────────┘
```

- **UI 요소**:

| 요소명 | 유형 | 설명 | 동작 |
|--------|------|------|------|
| 서비스 로고 | 이미지/텍스트 | RetroLens 로고 표시 | 없음 |
| 이메일 입력 | 입력 필드 | 이메일 주소 입력 | 이메일 형식 실시간 유효성 검사 |
| 비밀번호 입력 | 입력 필드 | 비밀번호 입력 (마스킹) | 없음 |
| 로그인 버튼 | 버튼 (Primary) | 인증 요청 | 성공 시 SCR-001(홈) 또는 SCR-000-2(팀 합류)로 이동, 실패 시 에러 메시지 |
| 회원가입 링크 | 텍스트 링크 | "계정이 없으신가요? 회원가입" | SCR-000-1(회원가입)로 이동 |
| 에러 메시지 영역 | 텍스트 (조건부) | 로그인 실패 시 원인 표시 | "이메일 또는 비밀번호가 올바르지 않습니다" |

- **관련 기능**: FEAT-000
- **이탈 경로**: SCR-001(홈), SCR-000-1(회원가입), SCR-000-2(팀 합류 - 팀 미배정 사용자)

│                                     │

## 4. 화면별 사용자 시나리오

### 4.1 시나리오: 신규 가입 및 팀 합류

**정상 흐름:**
1. 사용자가 서비스 URL에 접근하면 SCR-000(로그인) 화면이 표시된다
2. "계정이 없으신가요? 회원가입" 링크를 클릭하여 SCR-000-1(회원가입)로 이동한다
3. 이름, 이메일, 비밀번호를 입력하고 회원가입 버튼을 클릭한다
4. 가입 완료 메시지와 함께 SCR-000(로그인)으로 이동한다
5. 이메일과 비밀번호로 로그인한다
6. 팀 미배정 상태이므로 SCR-000-2(팀 합류)로 리다이렉트된다
7. 팀장에게 받은 팀 코드를 입력하고 합류 버튼을 클릭한다
8. 합류 성공 후 SCR-001(홈)이 표시된다

**예외 흐름 - 이메일 중복:**
1. 회원가입 시 이미 등록된 이메일을 입력한다
2. "이미 등록된 이메일입니다" 에러 메시지가 표시된다
3. 다른 이메일을 입력하거나, 로그인 링크를 클릭한다

**예외 흐름 - 잘못된 팀 코드:**
1. SCR-000-2(팀 합류)에서 잘못된 팀 코드를 입력한다
2. "유효하지 않은 팀 코드입니다" 에러 메시지가 표시된다
3. 올바른 코드를 다시 입력한다

---

### 4.2 시나리오: 회고 세션 생성 (팀장)

**정상 흐름:**
1. 팀장이 SCR-001(홈)에 진입한다
2. "세션 생성" 버튼을 클릭하여 SCR-002(세션 생성)로 이동한다
3. 세션 제목(예: "Sprint 12 회고"), 프로젝트명을 입력한다
4. 회고 기간을 선택한다 (선택 사항)
5. 생성 버튼을 클릭한다
6. 세션이 생성되고 고유 URL이 발급된다
7. URL을 복사하여 팀원에게 공유한다
8. SCR-003(피드백 입력) 화면으로 이동한다

**예외 흐름 - 필수 입력 누락:**
1. 세션 제목을 입력하지 않고 생성 버튼을 클릭한다
2. "세션 제목을 입력해주세요" 유효성 검사 메시지가 표시된다

---

### 4.3 시나리오: 익명 피드백 작성 (팀원)

**정상 흐름 (AI 문체 변환 OFF):**
1. 팀원이 SCR-001(홈)에서 진행중인 세션을 클릭하여 SCR-003(피드백 입력)에 진입한다
2. 세션 정보(제목, 프로젝트명)와 익명성 안내를 확인한다
3. "Problem" 탭을 선택한다
4. "배포할 때마다 30분씩 걸리는 거 진짜 미치겠음... ㅠㅠ"라고 입력한다
5. AI 문체 변환 토글이 OFF 상태인 것을 확인한다
6. 제출 버튼을 클릭한다
7. "피드백이 제출되었습니다" 확인 메시지가 표시된다
8. 피드백 카운트가 업데이트된다

**정상 흐름 (AI 문체 변환 ON):**
1. 팀원이 SCR-003(피드백 입력)에서 피드백을 입력한다
2. AI 문체 변환 토글을 ON으로 전환한다
3. 제출 버튼을 클릭한다
4. SCR-003-1(AI 문체 변환 미리보기 모달)이 표시된다
5. 원본: "배포할 때마다 30분씩 걸리는 거 진짜 미치겠음... ㅠㅠ"
6. 변환: "배포 프로세스에 약 30분이 소요되어 개선이 필요합니다."
7. 변환 결과를 확인하고 제출 버튼을 클릭한다
8. 피드백이 변환된 텍스트로 제출되고, 모달이 닫힌다

**예외 흐름 - 변환 결과 불만족:**
1. AI 문체 변환 미리보기에서 변환 결과를 확인한다
2. 의미가 충분히 전달되지 않는다고 판단한다
3. "재변환" 버튼을 클릭한다
4. AI가 새로운 변환 결과를 생성한다
5. 만족스러운 결과를 확인하고 제출한다

**예외 흐름 - AI 변환 실패:**
1. AI 문체 변환 모달에서 변환 처리 중 오류가 발생한다
2. "문체 변환에 실패했습니다. 원문으로 제출하시겠습니까?" 안내가 표시된다 [가정]
3. 사용자가 원문 제출을 선택하거나 취소한다

**예외 흐름 - 마감된 세션:**
1. 이미 마감된 세션의 URL로 접근한다
2. "이 세션은 마감되었습니다" 안내가 표시된다
3. 피드백 입력 필드와 제출 버튼이 비활성화된다

## 7. 반응형 레이아웃 설계

Tailwind CSS 기본 브레이크포인트 사용: `md` = 768px.

### 7.1 레이아웃 전환 구조

```
모바일 (< 768px)                  데스크탑 (≥ 768px)
─────────────────                 ──────────────────────────────────
┌──────────────────┐              ┌────────┬─────────────────────────┐
│ ☰ RetroLens (탑바) │              │        │                         │
├──────────────────┤              │  사이드  │      메인 콘텐츠          │
│                  │              │  바     │                         │
│    메인 콘텐츠    │              │  (256px)│    (flex-1, min-w-0)    │
│                  │              │  fixed  │                         │
└──────────────────┘              └────────┴─────────────────────────┘
  ← 사이드바: 슬라이드 오버레이        ← 사이드바: sticky 고정
```

### 7.2 GNB 반응형 동작

| 상황 | 모바일 동작 | 데스크탑 동작 |
|------|------------|--------------|
| 초기 상태 | 사이드바 숨김 (`-translate-x-full`) | 사이드바 항상 표시 (`translate-x-0`) |
| 햄버거 클릭 | 사이드바 슬라이드 인 (`translate-x-0`) | 해당 없음 |
| 오버레이 클릭 | 사이드바 슬라이드 아웃 | 해당 없음 |
| 메뉴 항목 클릭 | `watch(route.path)` → 자동 닫힘 | 해당 없음 |
| 상단 탑바 | 표시 (`md:hidden`) | 숨김 |
| 사이드바 너비 | `w-56` (224px), `z-40`, `fixed` | `w-56` (224px), `sticky` |

**모바일 오버레이:** 사이드바 열림 시 `bg-black/40` 반투명 배경 추가 (`z-30`). 배경 클릭으로 닫기 가능.

**CSS transform 활용:** DOM 추가/제거 없이 `transition-transform duration-200`으로 슬라이드. reflow 없는 GPU 가속 애니메이션.

### 7.3 페이지별 반응형 처리

| 화면 | 모바일 | 데스크탑 |
|------|--------|----------|
| SCR-001 홈 | 세션 카드 1컬럼, 전체 너비 | 카드 리스트, max-w-5xl |
| SCR-003 피드백 입력 | K/P/T 탭 전체 너비, 텍스트영역 전체 너비 | 컨테이너 중앙 정렬 |
| SCR-004 대시보드 | K/P/T 섹션 수직 스택 (`flex-col`) | 3컬럼 그리드 (`grid-cols-3`) |
| SCR-005 공유 인사이트 | 카드 1컬럼 | 카드 리스트, max-w-5xl |
| 모달 전체 | `fixed inset-0` 전체 화면 활용 | `max-w-md` 중앙 모달 |

### 7.4 스크롤바 스타일링

크로스 브라우저 커스텀 스크롤바 (thin, 녹색 계열):

```css
scrollbar-width: thin;
scrollbar-color: #d4ddd0 transparent;  /* Firefox */
::-webkit-scrollbar { width: 6px; }   /* Chrome/Safari */
::-webkit-scrollbar-thumb { background-color: #d4ddd0; border-radius: 3px; }
```
