## 3. 상세 기능

### FEAT-000: 인증 (로그인/회원가입)

- **설명:** 사용자가 이메일과 비밀번호로 회원가입, 로그인, 로그아웃할 수 있으며, 인증 후 세션이 유지된다. 팀 미배정 사용자는 팀 합류 화면으로 안내된다.
- **입력:** 이메일, 비밀번호, 이름 (회원가입 시)
- **출력:** 세션 토큰 발급, 인증 상태에 따른 화면 라우팅
- **수용 기준:**
  - 회원가입 후 즉시 로그인이 가능하다
  - 인증 후 세션 토큰이 발급되며 자동 만료가 적용된다
  - 인증되지 않은 사용자는 API에 접근할 수 없다
  - 팀 미배정 사용자는 팀 합류 화면(팀 코드 입력 또는 초대 수락)만 접근 가능하다
  - 비밀번호는 최소 8자, 영문+숫자 조합이어야 한다
  - 최고관리자/팀장/팀원 3단계 권한이 적용된다
- **관련 요구사항:** REQ-U006, REQ-B005

### FEAT-001: 회고 세션 생성

- **설명:** 팀장이 새로운 회고 세션을 생성하면 고유 URL이 발급되고, 팀원들이 해당 URL로 접근하여 피드백을 입력할 수 있다.
- **입력:** 세션 제목, 프로젝트명, 회고 기간(선택)
- **출력:** 고유 세션 URL, 세션 상세 페이지
- **수용 기준:**
  - 세션 생성 후 고유 URL이 발급된다
  - 발급된 URL로 접근 시 피드백 입력 화면이 표시된다
  - 생성된 세션이 세션 목록에서 조회된다
  - 팀장 이상이 세션을 종료(마감)할 수 있다
- **관련 요구사항:** REQ-U005, REQ-B003

### FEAT-002: 익명 피드백 입력

- **설명:** 팀원이 K/P/T(Keep/Problem/Try) 카테고리를 선택하여 익명으로 피드백을 제출한다. 서버에서도 작성자를 추적할 수 없다.
- **입력:** 카테고리(Keep/Problem/Try), 피드백 내용
- **출력:** 제출 완료 확인 메시지
- **수용 기준:**
  - 로그인한 팀원만 본인 팀 세션에 피드백을 제출할 수 있다
  - 피드백 저장 시 작성자 정보와 연결되지 않는다 (서버에서도 추적 불가)
  - 카테고리별로 여러 개의 피드백을 입력할 수 있다
  - 제출 후 수정/삭제가 불가하다 (익명성 보장)
- **관련 요구사항:** REQ-U001, REQ-U002, REQ-U009, REQ-B001

### FEAT-002-1: AI 문체 변환 (익명성 강화)

- **설명:** 피드백 제출 시 AI가 개인 특유의 말투, 이모지, 표현을 중립적으로 변환하여 문체 기반 작성자 식별을 차단한다.
- **입력:** 원본 피드백 텍스트, 문체 변환 ON/OFF 토글
- **출력:** 변환된 피드백 텍스트 미리보기, 최종 제출 확인
- **수용 기준:**
  - 문체 변환 ON/OFF 토글이 제공된다
  - 변환 전/후 비교 미리보기가 표시된다
  - AI가 원래 의미를 유지하면서 개인 특유의 표현만 중립화한다
  - 사용자가 변환 결과를 확인한 후 제출할 수 있다
  - 변환 결과가 불만족스러운 경우 재요청이 가능하다 [가정: PRD 10.1에서 "재요청 가능" 언급]
- **관련 요구사항:** REQ-U001, REQ-B001

### FEAT-003: 실시간 참여 현황

- **설명:** 현재 세션의 피드백 제출 현황을 실시간으로 표시하여 팀장이 참여율을 확인하고 독려할 수 있다.
- **입력:** 없음 (세션 컨텍스트에서 자동 수집)
- **출력:** 총 피드백 수, 카테고리별 피드백 수 (실시간 업데이트)
- **수용 기준:**
  - 새로운 피드백이 제출되면 카운트가 자동으로 업데이트된다
  - 카테고리(K/P/T)별 피드백 수가 각각 표시된다
  - 참여자 개인 식별 정보는 노출되지 않는다 [가정: 익명성 원칙 적용]
- **관련 요구사항:** REQ-U003, REQ-B001

### FEAT-004: 팀 대시보드

- **설명:** 회고 세션의 전체 피드백을 K/P/T 카테고리별로 구분하여 한눈에 조회할 수 있는 대시보드를 제공한다.
- **입력:** 세션 ID
- **출력:** Keep/Problem/Try 카테고리별 피드백 목록
- **수용 기준:**
  - 카테고리별 탭 또는 섹션으로 구분되어 표시된다
  - 각 피드백의 전체 내용이 표시된다
  - 카테고리별 피드백 수 카운트가 표시된다
- **관련 요구사항:** REQ-U003, REQ-B003

### FEAT-005: AI 인사이트 생성

- **설명:** 수집된 전체 피드백을 AI가 분석하여 핵심 이슈를 추출하고, 각 이슈에 대한 권장 액션을 도출한다.
- **입력:** 세션의 전체 피드백 데이터
- **출력:** 요약문, 핵심 이슈 목록(3~5개), 각 이슈별 권장 액션
- **수용 기준:**
  - "인사이트 생성" 버튼 클릭 시 AI 분석이 실행된다
  - 결과가 30초 이내에 표시된다
  - 핵심 이슈가 3~5개 추출된다
  - 각 이슈에 대한 구체적인 권장 액션이 제시된다
  - 팀장 권한만 인사이트 생성 버튼을 사용할 수 있다
- **관련 요구사항:** REQ-U004, REQ-B002

### FEAT-006: 피드백 투표 (실명)

- **설명:** 이미 공개된 피드백에 대해 로그인한 사용자가 공감 투표를 할 수 있다. 투표는 실명으로 기록된다.
- **입력:** 피드백 ID, 투표 액션(투표/취소)
- **출력:** 업데이트된 투표 수
- **수용 기준:**
  - 피드백당 1인 1표가 보장된다
  - 투표 수 기준으로 피드백을 정렬할 수 있다
  - 투표를 취소할 수 있다
  - 로그인한 사용자만 투표 가능하다
- **관련 요구사항:** REQ-U007

### FEAT-007: 인사이트 공유 (팀장 전용)

- **설명:** 팀장이 AI로 생성된 인사이트를 다른 팀에 공유하거나 비공유로 전환할 수 있다. 공유 시 피드백 원문은 비공개된다.
- **입력:** 인사이트 ID, 공유/비공유 토글
- **출력:** 공유 상태 변경 확인, 공유된 인사이트 목록 페이지
- **수용 기준:**
  - 공유/비공유 토글이 제공된다
  - 공유 시 다른 팀원들이 인사이트만 조회 가능하다 (피드백 원문 비공개)
  - 공유된 인사이트 목록 페이지가 제공된다
  - 비공유 전환 시 해당 팀원만 조회 가능하다
  - 팀장 이상만 공유 설정을 변경할 수 있다
- **관련 요구사항:** REQ-B006

### FEAT-008: 액션 아이템 생성

- **설명:** Problem 카테고리의 피드백을 기반으로 액션 아이템을 생성하고 담당자와 기한을 지정하여 추적한다.
- **입력:** 피드백 ID, 액션 아이템 내용, 담당자, 기한
- **출력:** 생성된 액션 아이템, 상태 업데이트
- **수용 기준:**
  - 피드백에서 액션 아이템으로 전환할 수 있다
  - 담당자와 기한을 설정할 수 있다
  - 완료/미완료 상태를 관리할 수 있다
- **관련 요구사항:** REQ-B004

### FEAT-009: 히스토리 비교

- **설명:** 이전 회고 세션들과 현재 세션의 트렌드를 비교하여 반복되는 이슈를 발견한다.
- **입력:** 현재 세션 ID
- **출력:** 최근 5개 세션 비교 결과, 반복 이슈 하이라이트
- **수용 기준:**
  - 최근 5개 세션을 비교할 수 있다
  - 반복되는 이슈가 하이라이트로 표시된다
- **관련 요구사항:** REQ-U008, REQ-B002

### FEAT-010: 팀 관리 (최고관리자)

- **설명:** 최고관리자가 팀을 생성/수정/삭제하고, 팀원을 이동/제거하며, 팀장을 지정/변경한다.
- **입력:** 팀 정보(이름 등), 팀원 정보, 팀장 지정 대상
- **출력:** 팀 목록, 팀 상세 정보
- **수용 기준:**
  - 팀 CRUD가 가능하다
  - 팀원을 이동/제거할 수 있다
  - 팀장을 지정/변경할 수 있다
  - 최고관리자만 접근 가능하다
- **관련 요구사항:** REQ-B006

### FEAT-011: 사용자 관리 (최고관리자)

- **설명:** 최고관리자가 사용자의 역할(ADMIN/LEADER/MEMBER)을 변경하고 계정을 활성화/비활성화한다.
- **입력:** 사용자 ID, 변경할 역할 또는 상태
- **출력:** 업데이트된 사용자 정보
- **수용 기준:**
  - 역할 변경(ADMIN/LEADER/MEMBER)이 가능하다
  - 계정 활성화/비활성화가 가능하다
  - 최고관리자만 접근 가능하다
- **관련 요구사항:** 해당 없음 (관리 기능)

### FEAT-012: 사내 전체 통계 (최고관리자)

- **설명:** 최고관리자 전용으로 전체 팀별 회고 빈도, 참여율, 반복 이슈 트렌드 등 통합 통계를 제공한다.
- **입력:** 조회 기간(선택), 필터 조건(선택)
- **출력:** 팀별 회고 빈도, 참여율, 사내 전체 반복 이슈 트렌드, 팀간 비교 분석
- **수용 기준:**
  - 팀별 회고 빈도와 참여율이 표시된다
  - 사내 전체 반복 이슈 트렌드가 표시된다
  - 팀간 비교 분석이 가능하다
  - 최고관리자만 접근 가능하다
- **관련 요구사항:** REQ-B002

### FEAT-013: 슬랙 연동

- **설명:** 회고 시작 알림과 결과 요약을 슬랙으로 공유한다.
- **입력:** 슬랙 채널 설정, 알림 타입
- **출력:** 슬랙 메시지 발송
- **수용 기준:**
  - 회고 세션 생성 시 슬랙 알림이 전송된다
  - 인사이트 생성 후 요약이 슬랙에 공유된다
  - [가정] 슬랙 웹훅 또는 OAuth 기반 연동
- **관련 요구사항:** REQ-B003

### FEAT-014: 감정 분석

- **설명:** 피드백의 톤을 AI가 분석하여 긍정/부정/중립으로 분류한다.
- **입력:** 피드백 텍스트
- **출력:** 감정 분류 결과(긍정/부정/중립), 감정 분포 시각화
- **수용 기준:**
  - 각 피드백에 감정 태그가 표시된다
  - 세션 전체의 감정 분포가 시각화된다
  - [가정] 대시보드에 감정 분석 섹션 추가
- **관련 요구사항:** REQ-B002

### FEAT-015: 내보내기

- **설명:** 회고 결과를 PDF, 노션, 컨플루언스 등 외부 형식으로 내보낸다.
- **입력:** 세션 ID, 내보내기 형식 선택
- **출력:** 선택한 형식의 문서 파일 또는 외부 서비스 연동 결과
- **수용 기준:**
  - PDF 형식으로 내보내기가 가능하다
  - 노션/컨플루언스 연동이 가능하다
  - [가정] 내보내기 결과에 피드백, 인사이트, 액션 아이템이 포함된다
- **관련 요구사항:** REQ-B003

## 4. 우선순위

| **P0** | FEAT-001 | 회고 세션 생성 | 피드백을 수집할 컨테이너가 없으면 서비스 자체가 성립하지 않음 |
| **P0** | FEAT-002 | 익명 피드백 입력 | "솔직한 피드백 수집"이라는 제품의 존재 이유 그 자체 |
| **P0** | FEAT-002-1 | AI 문체 변환 | 제품의 핵심 차별점. 이 기능 없이는 기존 도구 대비 경쟁력 부족 |
| **P0** | FEAT-004 | 팀 대시보드 | 수집된 피드백을 확인하는 유일한 수단. 없으면 피드백 수집 의미가 없음 |
| **P0** | FEAT-005 | AI 인사이트 생성 | "AI 패턴 분석"이라는 두 번째 핵심 가치. 단순 수집 도구와 차별화하는 핵심 |
| **P1** | FEAT-003 | 실시간 참여 현황 | 참여 독려에 유용하나 MVP 핵심 플로우 완성에 필수는 아님 |
| **P1** | FEAT-006 | 피드백 투표 | 피드백 우선순위화에 도움되지만, 첫 출시 없이도 인사이트 도출 가능 |
| **P1** | FEAT-007 | 인사이트 공유 | 팀 간 학습이라는 확장 가치. 단일 팀 내에서는 없어도 동작 |
| **P1** | FEAT-008 | 액션 아이템 생성 | 회고 실효성 강화에 중요하나, MVP에서는 인사이트 도출까지만으로 가치 제공 가능 |
| **P1** | FEAT-009 | 히스토리 비교 | 세션이 누적되어야 의미가 있으므로 초기에는 활용도 낮음 |
| **P1** | FEAT-010 | 팀 관리 (최고관리자) | 3단계 권한 체계에 필수. 바로 구현 |
| **P1** | FEAT-011 | 사용자 관리 (최고관리자) | 최고관리자 전용 관리 기능. 바로 구현 |
| **P1** | FEAT-012 | 사내 전체 통계 (최고관리자) | 최고관리자 전용 분석 기능. 바로 구현 |
| **P2** | FEAT-013 | 슬랙 연동 | 외부 서비스 의존. 제품 안정화 후 사용자 요구에 따라 추가 |
| **P2** | FEAT-014 | 감정 분석 | 부가 분석 기능. 핵심 인사이트 생성과 독립적 |
| **P2** | FEAT-015 | 내보내기 | 부가 유틸리티. 대시보드에서 직접 확인하는 것으로 대체 가능 |

### PM 검증 의견

PRD의 우선순위 분류는 전반적으로 타당하다. 특히 FEAT-002-1(AI 문체 변환)을 P0로 설정한 것은 제품의 핵심 차별점이므로 적절하다. 다만, 다음 사항을 참고할 필요가 있다:

- **FEAT-002-1 리스크:** AI API 의존도가 높아 API 장애 시 MVP 전체가 영향받을 수 있다. PRD에서 "하드코딩된 샘플 인사이트로 대체"라는 대응 방안을 제시했으나, 문체 변환에 대한 폴백 전략도 필요하다. [가정: 문체 변환 실패 시 원문 그대로 제출하되 경고 표시]
- **FEAT-008 재고:** 액션 아이템 추적은 "회고의 실효성"이라는 제품 목표(완료율 70%)와 직결되므로, 여건이 되면 P1 내에서도 높은 우선순위로 검토 필요
